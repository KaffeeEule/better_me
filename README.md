# better_me
Саморазвитие без выгорания. Пройди тест, узнай свою психологическую архитектуру и получай одну простую 5-минутную практику каждый день. Мы подобрали задания, которые усиливают твои таланты и мягко подталкивают к росту там, где это нужнее всего. Твоя лучшая версия — это серия маленьких шагов.

ехническое задание: Web-приложение «Daily Practice»
1. Общее описание
Минималистичное веб-приложение для ежедневного личностного развития. Система определяет психотип пользователя через тест и ежедневно предлагает одну практику, адаптированную под его профиль.

Стек: HTML5, CSS3, JavaScript (ES6+).
Хранение данных: localStorage (без серверной части).

2. Структура данных (JSON)
2.1. Контент (Задания)
Массив объектов, разделенный по ключам типов личности: explorer, zen_master, achiever, creator, altruist.

Как считать результаты

Каждому ответу присваивается тип личности:

А – Исследователь (высокая "Открытость опыту", поиск новизны)
Б – Созерцатель (низкий "Нейротизм", поиск равновесия и осознанности)
В – Стратег (высокая "Добросовестность", потребность в контроле)
Г – Творец (креативность, самовыражение через эстетику)
Д – Эмпат (высокая "Доброжелательность", социально-ориентированный)
Механика:

Мажоритарный метод: Считается количество ответов каждого типа. Преобладающий тип – основной.
Процентное соотношение: Например, 4 ответа "А" и 3 ответа "В" – Исследователь-Стратег (60% и 40% практик соответственно).
Нулевой результат: Равное распределение баллов – тип "Универсал" (микс из всех практик).

2.2. Состояние пользователя (State)
Объект, сохраняемый в localStorage:

userProfile: Обьект с весами типов (напр. { explorer: 0.4, ... }).

lastTaskDate: Дата последнего получения задания (в формате YYYY-MM-DD).

currentTask: Текст текущего задания.

usedTasks: Массив ID или текстов уже выполненных заданий (чтобы избежать повторов).

3. Логика работы приложения
Этап 1: Первичный тест (Onboarding)
Ввод: 10 вопросов с 5 вариантами ответов каждый.

Обработка: Каждому варианту соответствует один из 5 типов.

Результат: Суммирование ответов и перевод в проценты.

Пример: Если выбрано 4 ответа «Исследователь», вес типа = 0.4 (или 40%).

Запись: Сохранение профиля в localStorage.

Этап 2: Алгоритм выбора задания (90/10)
Приложение проверяет lastTaskDate. Если дата сменилась, запускается алгоритм выбора:

Определение «Зоны роста»: Найти тип личности с минимальным значением в userProfile.

Бросок кубика (Random): Генерируется число R от 0 до 1.

Условие 10% (Слабое место): Если R <= 0.1, выбирается случайное неиспользованное задание из массива «Зоны роста».

Условие 90% (Сильные стороны): Если R > 0.1, выбирается задание методом Weighted Random Selection (взвешенное случайное число) на основе профиля пользователя.

Механика: Типы с большим весом имеют пропорционально больший шанс выпасть в этом секторе.

Исключение дублей: Выбранное задание сверяется с массивом usedTasks. Если оно уже выполнялось, поиск повторяется.

5. Имплементация (Snippet для IDE)
Пример функции на JS для выбора задания по логике 90/10:

JavaScript
function getDailyTask(profile, allTasks, usedTasks) {
  const rand = Math.random();
  let targetType = '';

  if (rand <= 0.1) {
    // 10% - выбираем самое слабое место
    targetType = Object.keys(profile).reduce((a, b) => profile[a] < profile[b] ? a : b);
  } else {
    // 90% - взвешенный рандом по сильным сторонам
    const totalWeight = Object.values(profile).reduce((a, b) => a + b, 0);
    let threshold = Math.random() * totalWeight;
    
    for (const [type, weight] of Object.entries(profile)) {
      threshold -= weight;
      if (threshold <= 0) {
        targetType = type;
        break;
      }
    }
  }

  // Фильтрация неиспользованных и возврат задания
  const available = allTasks[targetType].filter(t => !usedTasks.includes(t));
  return available[Math.floor(Math.random() * available.length)];
}
6. Требования к реализации
Чистый JS: Избегать тяжелых библиотек для сохранения легкости.

Адаптивность: Корректное отображение на мобильных устройствах (viewport meta tag).

Безопасность: Проверка наличия данных в localStorage при каждой загрузке страницы (null check).
